# Lint

위키를 건강하게 유지한다. 고치지는 않고 **진단하고 제안**한다(사용자 승인 후 수정).

## 체크 항목

- **orphan 페이지** — 어떤 페이지에서도 `[[링크]]`되지 않는 페이지.
- **깨진 링크** — `[[slug]]`가 가리키는 페이지가 없음.
- **index 누락** — `wiki/`에 있으나 `index.md`에 없는 페이지(또는 그 반대).
- **모순** — 두 페이지가 같은 사실을 다르게 말함.
- **stale** — 오래된 `first_seen`/`ingested`인데 새 자료로 갱신 안 된 high-volatility 주제.
- **출처 없는 주장** — 정량 주장인데 단위/시점/출처가 없음(ANTIGRAVITY.md §5).
- **glossary 누락** — 본문에 쓰인 기술 용어가 `glossary.md`에 없음.
- **개념 후보** — 여러 페이지에서 언급되나 자기 페이지가 없는 중요한 개념.

## 절차

1. `index.md`와 `log.md`를 읽어 전체 모양을 파악한다.
2. `wiki/`를 훑어 위 항목을 점검한다(grep/파일 탐색으로 충분).
3. 발견을 **카테고리별 목록**으로 보고한다. 각 항목에 제안 조치를 단다.
4. 사용자가 고치라고 하면 고친다. 패턴이 3회 이상 반복되면 ANTIGRAVITY.md 스키마
   변경을 제안한다.

## 새 작업 제안

lint는 좋은 다음 질문과 찾아볼 자료를 제안하기 좋은 자리다. 위키가 커지도록
돕는다.
