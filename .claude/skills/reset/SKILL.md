# Reset

테스트 중 입력한 데이터를 지우고 위키를 배포 직후의 빈 상태로 되돌린다.

## ⚠️ 파괴적 작업 — 먼저 확인한다

이 작업은 `wiki/**`의 모든 페이지, `raw/clips/**`·`raw/uploads/**`의 모든
자료를 **삭제**하고 `index.md`·`log.md`·`glossary.md`를 `seed/`의 빈
스켈레톤으로 되돌린다. 되돌릴 수 없다(git에 커밋해 두지 않았다면).

**실행 전에 반드시 사용자에게 "정말 초기화할까요? 모든 위키 페이지와 ingest한
자료가 삭제됩니다."라고 묻고 명시적 동의를 받는다.**

## 절차

1. 사용자에게 확인을 받는다(위).
2. 무엇이 지워질지 간단히 보여준다 — `wiki/`의 페이지 수, `raw/clips`·
   `raw/uploads`의 파일 수.
3. 동의하면 리셋 스크립트를 실행한다:
   ```bash
   npm run reset
   ```
   (또는 `node scripts/reset.mjs`)
4. 스크립트 출력(무엇이 삭제/복원됐는지)을 사용자에게 보고한다.
5. Quartz serve가 실행 중이면 사이트가 자동으로 빈 스켈레톤으로 핫리로드된다.

## 부분 삭제 (선택)

사용자가 전체가 아니라 특정 자료만 지우길 원하면(예: "방금 넣은 그 기사
관련 페이지 지워줘"), 전체 reset 대신:
1. `wiki/sources/`에서 해당 자료 페이지를 찾아 그것이 만든/갱신한 페이지 목록을 본다.
2. 그 자료만으로 생긴 페이지를 지우고, 공유 페이지에서는 해당 자료 기여분만 제거한다.
3. `index.md`·`log.md`를 갱신한다.
이 경우 `scripts/reset.mjs`를 쓰지 않는다.
