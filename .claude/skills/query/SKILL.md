# Query

위키에 대고 질문에 답한다. 좋은 답변은 채팅에서 사라지지 않고 위키에 쌓인다.

## 절차

1. **`index.md`를 먼저 읽는다** — 관련 페이지를 찾는다.
2. **관련 위키 페이지를 드릴다운**해서 읽는다.
3. **load-bearing 주장은 인용된 `raw/` 자료까지** 읽어 확인한다. 위키 페이지는
   포인터이자 종합이지 증거가 아니다 — 핵심 사실은 1차 자료로 확인한다.
4. **출처와 함께 답한다**. 수치는 단위·시점·출처를 함께. 위키에 없는 내용은
   없다고 말하고, 필요하면 새 자료를 ingest하자고 제안한다.
5. **저장 제안**. 답변이 한 번 쓰고 버리기 아까운 비교·분석·발견이면
   `wiki/notes/`에 페이지로 저장할지 묻는다. 저장하면 `index.md`·`log.md`도 갱신.

## 형식

질문에 따라 답변 형식은 달라진다 — 짧은 문단, 비교 표, bullet 목록 등.
사용자가 원하면 더 긴 산출물(노트 페이지)로 만든다.

## 하지 말 것

- 위키에 근거 없는 내용을 지어내지 않는다. 모르면 모른다고 한다.
- 위키 페이지를 증거로 인용하지 않는다 — 그 아래 `raw/` 자료를 인용한다.
