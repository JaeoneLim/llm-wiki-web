# Ingest

자료 하나를 구조화된, 교차 링크된 위키 페이지로 바꾼다. 이것은 **대화형**
작업이다 — 사용자와 함께 꼬리 질문을 주고받으며 위키를 완성한다.

## 입력

다음 중 하나를 받는다.

- `raw/` 아래 경로 (예: `raw/uploads/some-report.pdf`)
- URL — 이 경우 먼저 페치해 `raw/clips/{YYYY-MM-DD}-{slug}.md`로 저장한다.
  헤더에 원본 URL과 페치 일자를 보존한다. (백엔드가 미리 클립해 둔 경우 그
  경로를 받는다.)

## 절차

1. **CLAUDE.md를 읽는다** (항상 — 익숙해 보여도). 규칙은 load-bearing.
2. **자료를 끝까지 읽는다** — 무엇이든 쓰기 전에. PDF가 크면 나눠 읽는다.
3. **현재 `index.md`를 읽는다** — 이미 무엇이 있는지 안다. 페이지 중복을
   피한다. 새 페이지를 만들기보다 기존 개체/개념 페이지 갱신을 우선한다.
4. **핵심을 사용자와 논의한다** — 파일을 건드리기 전에 5~10개 bullet로:
   - 이 자료가 다루는 개체·개념
   - 핵심 주장(수치는 단위·시점과 함께)
   - 갱신될 법한 기존 위키 페이지
   - 기존 페이지와 모순되는 내용
   이 단계에서 **꼬리 질문**을 던진다 — 사용자가 무엇을 강조하고 싶은지,
   어떤 각도로 정리할지, 모르는 맥락이 있는지. 위키는 함께 만든다.
5. **페이지 연산을 번호 목록으로 제안한다**. 예:
   - "1. CREATE `wiki/sources/2026-06-02-foo-article.md`"
   - "2. CREATE `wiki/concepts/vector-embedding.md`"
   - "3. UPDATE `wiki/entities/openai.md` — 새 제품 추가"
   - "4. UPDATE `index.md`, `log.md`"
6. **사용자 확인을 기다린다**. 한 ingest는 보통 3~10개 파일을 건드린다 —
   사용자가 모양을 먼저 승인해야 한다. `1,3 yes; skip 2` 같은 압축 답변도 수용.
7. **승인된 연산을 실행한다**. 기존 파일은 `Edit`, 새 파일은 `Write`.
8. **`wiki/sources/`에 자료 요약 페이지를 만든다** — 이 자료가 만들거나 갱신한
   모든 페이지로의 링크를 담아, 자료↔지식의 양방향 추적을 만든다.
9. **레포 루트 `index.md`를 갱신한다** — 새 항목을 한 줄 요약과 함께 카테고리에
   추가. (주의: `wiki/index.md`를 새로 만들지 말 것. index/log/glossary는 항상
   레포 루트 파일이다 — CLAUDE.md §9.)
10. **레포 루트 `log.md`에 추가한다**:
    ```
    ## [YYYY-MM-DD] ingest | {자료 제목}

    - source: raw/clips/2026-06-02-foo.md
    - created: wiki/sources/..., wiki/concepts/...
    - updated: wiki/entities/...
    - notes: 미해결 질문이나 리스크
    ```
11. **레포 루트 `glossary.md`를 갱신한다** — 본문에 처음 등장하는 모든 기술 용어를
    한글+영문으로 추가(CLAUDE.md §8).
12. **언어 정책 준수**(CLAUDE.md §8): 본문 한글, 기술 용어 첫 등장 한글+영문,
    1차 자료 인용 원어 유지, 파일명/슬러그 영어 kebab-case.

## 규칙 (CLAUDE.md — 완화하지 않는다)

- **출처 추적**(§5). 사실 주장은 가능한 한 `raw/` 자료로 추적되게 한다.
  일반 지식으로 채운 부분은 그렇게 표시한다.
- **Eviction**(§6). 새 자료가 기존 주장을 대체하면 옛 페이지를 삭제하지 말고
  `status: superseded`로 표시한다.
- **중복 금지**. 같은 개체/개념의 페이지를 두 번 만들지 않는다 — 먼저 index를 본다.

## 하지 말 것

- 발행용 산문(블로그 글)을 쓰지 않는다. 위키는 블로그가 아니다.
- 이 레포 밖의 파일을 수정하지 않는다.
- §6의 사용자 확인 단계를 건너뛰지 않는다 — ingest는 여러 파일을 건드린다.
